Testing Workflow:

This follows the same structure as `a1`, but the expected outputs are generated by a Python AVL reference implementation.

1. Add or edit test cases in `all_tests.txt`
2. Run `bash generate.sh`
3. Run `python3 generate.py`
4. Run `bash test.sh`
5. Run `bash compare.sh`

Folder layout:

- `all_tests.txt`: master list of one-line test cases
- `tests/`: one generated input file per test
- `expected/`: expected output generated from the Python AVL reference
- `outputs/`: output produced by your `a2.cpp`

Input format:

Each line in `all_tests.txt` is one full program input. Operations are space-separated.

Examples:

- `A1 A2 A3 IN`
- `A1 A2 A3 PRE`
- `A1 D1 POST`

Supported operations:

- `A<number>`: insert
- `D<number>`: delete
- `PRE`: print preorder traversal
- `IN`: print inorder traversal
- `POST`: print postorder traversal

Notes:

- You can include more than one traversal in a single test line if you want multiple output lines.
- Blank lines and lines starting with `#` are ignored by `generate.sh`.
